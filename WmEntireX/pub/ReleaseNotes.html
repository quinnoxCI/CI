Release Notes

webMethods EntireX Adapter Version 9.12.0.0

The webMethods EntireX Adapter enables the webMethods Integration Server to consume (IS outbound) as well as provide (IS inbound) EntireX RPC services.

EntireX supports bidirectional integration with Natural, COBOL, PL/I, C and other applications on z/OS, z/VSE, BS2000 and other platforms.

Moreover, EntireX supports unidirectional integration with webMethods ApplinX, IBM IMS Connect, and IBM CICS® ECI.

For a complete overview of supported connectivity, see the webMethods EntireX documentation at http://documentation.softwareag.com.

Connectivity to EntireX RPC Servers

The EntireX Adapter allows you to create connections based on the interface definitions of mainframe programs. The connected application programs are exposed to the Integration Server through EntireX RPC technology and RPC servers.

Calling a Mainframe Program in a Flow (IS Outbound)

With the EntireX Adapter a flow is enabled to call a mainframe program using an adapter service and an adapter connection. The connection is configured to route the RPC calls to an EntireX RPC server, also for Reliable RPC. In this scenario, the EntireX Adapter acts as an RPC client.

In addition, mainframe programs running under IBM IMS can be called using IMS Connect. Mainframe program under IBM CICS can be called using CICS ECI.

Calling a Flow from a Mainframe Program (IS Inbound)

For the opposite direction, that is, calling an IS flow, the EntireX Adapter acts as an RPC server. This exposes services of the Integration Server by means of RPC to mainframe programs. These mainframe programs act as RPC clients and call the services. The RPC clients can also use Reliable RPC. Within webMethods Integration Server, listeners and adapter notifications are used to subsequently call desired target services.

Using Eclipse to Create Connections at Design Time

With the EntireX Workbench, you can generate all technical assets necessary to call a service, for example you can extract the interface definition of the program to call. Use this interface definition to create an adapter connection and the corresponding adapter services.

Also, interface definitions can be extracted from Integration Server packages. With these definitions, you can generate all Integration Server nodes, including connections and listeners.

The EntireX Workbench and the EntireX Adapter add-on are both Eclipse-based.
